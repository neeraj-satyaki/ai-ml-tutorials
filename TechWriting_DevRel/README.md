# Technical Writing + DevRel

Writing → blogging → talks → community → devrel practice.

## Technical writing
- Audiences: API docs, tutorials, conceptual, reference, guide.
- **Diátaxis** documentation framework (Tutorials / How-To / Reference / Explanation).
- Style guides: Microsoft, Google Developer Docs.
- Plain-language principles (Hemingway-esque).
- Information architecture.
- **Docs-as-Code**: Markdown + MkDocs / Docusaurus / Starlight / Read the Docs.
- Diagrams: draw.io, Excalidraw, Mermaid, PlantUML.
- Versioning docs per release.
- API reference generated from OpenAPI.
- Release notes + changelog (Keep a Changelog format).

## Blogging + content
- Topics: evergreen vs timely.
- Story-centered technical narratives.
- Runnable, correct code samples.
- Basic SEO for developers.
- Newsletters (Substack, Beehiiv, Buttondown).
- Community engagement.
- User-focused writing.
- DevRel vs marketing boundary.

## Talks + teaching
- Winning **CFPs** (titles, abstracts, hooks).
- Talk structure: problem → solution → demo.
- Live coding — risk mitigation (fallback, checkpoints, small diffs).
- Recording: OBS, ScreenFlow, Loom.
- Feynman technique for teaching.
- Workshop design + labs.
- Teaching rubrics.

## Community management
- Discord / Slack community patterns.
- GitHub issue hygiene.
- **Code of Conduct**. Moderation policy.
- OSS governance (BDFL, meritocratic, foundation).
- Mentorship programs.
- Developer advocate skills: listening, empathy, bridging eng ↔ external devs.
- Hackathons.

## DevRel practice
- DevRel personas: Dev Marketer, Developer Advocate, Developer Educator.
- **DevRel metrics**: pipeline, adoption, DX.
- Developer Experience (DX) — time-to-first-hello-world.
- Onboarding tracks / tutorial series.
- Developer journey map.
