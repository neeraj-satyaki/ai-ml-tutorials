# Open Source Contribution

From first PR to maintainer at scale.

## First contributions
- **Find issues**: "good first issue", "help wanted", "up-for-grabs", "goodfirstissue.com".
- Read `CONTRIBUTING.md` + `CODE_OF_CONDUCT.md` before touching code.
- Small focused PR. Title + description explain *why*, not just *what*.
- Respond to review promptly + respectfully.
- Tests required — no exceptions.
- **CLA vs DCO** (Developer Certificate of Origin `Signed-off-by`).

## Maintainership
- Triage issues (labels, duplicates, reproduce).
- Automation (Stale bot, Dependabot, release-drafter).
- **Release management + SemVer**. Conventional Commits.
- Changelog (**Keep a Changelog** style).
- Governance models: BDFL, meritocratic, foundation (ASF, CNCF).
- Maintainer burnout — real; delegate + say no.
- Funding: GitHub Sponsors, Open Collective, Tidelift, Polar.

## Project health
- README best practices (what / install / quickstart / link to docs / badges).
- `SECURITY.md`, `CODEOWNERS`, `ISSUE_TEMPLATE/`, `PULL_REQUEST_TEMPLATE.md`.
- GitHub Discussions for Q&A.
- **ADRs** (Architecture Decision Records).
- **SBOM** for OSS supply chain.
- Reproducible builds (Nix, Bazel).

## Licensing
- **Permissive**: MIT, Apache 2.0 (patent grant), BSD-2/3.
- **Copyleft**: GPL, LGPL, AGPL (network clause).
- Controversial: **SSPL** (MongoDB), **BUSL** (HashiCorp → renamed Licensing), **FairCode**.
- License compatibility matrix — you can combine some, not others.
- DCO vs CLA trade-offs.
- Attribution notices (`NOTICE` file under Apache).
- Trademark policies (separate from copyright).
